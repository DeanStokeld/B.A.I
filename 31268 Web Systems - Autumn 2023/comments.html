Webpage Structure

1. Header

The header div (highlighted in  yellow ) contains the tityle of the website. It uses the <h1> tag and it has the class 'header' which is called as .header in websystems.css file.



2. Navigation Bar

The navigation bar div (highlighted in  red ) contains the navigation bar of the website, which is visible on all pages. It uses the <a> tag (which is used to create hyperlinks) and it has the class 'nav' which is called as .nav in the websystems.css file. The navigation bar is a large block of hyperlinks to the pages 'Home' (index.html), Past (past.html), Future (future.html) and Comments (comments.html).


3. Main

The main div (highlighted in  blue , which is also referred to as 'body') is where all of the information of the selected page is contained. This area contains several components such as header tags (<h1>, <h2>, <h3> & <h4>), paragraph tags (<p>), image tags (<img>)and table tags (<table>) in the HTML code to assist in presenting several different types of information to the user.

On pages 'Past' and 'Comments' (this page), I have inserted a small icon on the bottom right with the text 'Back to top', once clicked this allows the user to immediately jump to the top of the page instead of having to scroll all the way back up.


4. Footer

The footer div (highlighted in  green ) is located at the bottom of the screen and provides some information regarding copyright information, contact information, links to other pages or resources, and other information. In the footer on this page I have copyright information and other general information such as the Subject name of this course and the univerisyt.

Towards the right side of the footer I have small hyperlinks separated by a vertical bar that links to the other pages of the website.


Aesthetics

The approach of this website is to take a minimalistic approach to display information in a simplistic manner. This approach encourages accessability which allows readability of the webpage. There are webpages that do not take this approach, such as Pacific Northwest X-Ray Inc and Ling's Cars.

Colour

The colours used in this webpage is black, white and gray. I chose this grayscale colour palette to prevent any affect on the readability of the webpage. This also enables people with colour-blindness to browse the website without any difficulty. However, I couldn't avoid having colour in some images.


Font

The font chosen for this website is the font family of Arial, Helvetica and sans-serif. A font family is a list of fonts that the webpage will prioritise from left to right until it finds a font that is available to be used.


Interactivity

There are two parts of this webpage that incorporates some interactivity. The first being the navigation bar, using the :hover and :active tags in CSS, I was able to change the background colour and font colour of the navigation bar once hovered over with a mouse and/or clicked (visited).
Another area of interactivity is the button on the bottom right of your screen now, the 'Back to Top' brings the user to the top of the webpage once clicked.


Accessibility

The W3C Web Accessibility Initiate has outlined guidelines for how to successfully achieve website accessibility. They believe accessibility is essential for every website as without it you cannot create a high quality websites. They believe that website accessability, specifically the ability to access information, as a basic human right. This has also been outlined by the UN as well, which you can read here if you like.
For the reasons listed above, I have considered the following aspects of this website:

  • Font Sizing
    • The font sizing on this webpage is relative to how much the user is zoomed in. For example, if the user zooms in the font size will increase.

  • Screen Readers
    • Screen readers are unable to relay information from an image to the user, however using the 'alt' tag in the <img> tag I am able to described the image which a screen reader will be able to read and relay that information to the user.

  • Font Type
    • The font selected for this website set as a family. As described in Aesthetics, the font family chosen contains the following fonts: Arial, Helvetica and sans-serif. The webpage will prioritise the fonts from left to right until an available font is found. These font is very legible and will present the users the information without difficulty.

  • W3C Valiator Checker
    • I ran all of my HTML code through the The W3C Markup Validation Service and received no errors or warnings. I also ran my CSS through The W3c CSS Valiation Service and recieved zero errors or warnings.

  • Mobile Users
    • Mobile users were also considered while developing this website. With this in mind, I included a media query that targets devices with a maximum width of 1000 pixels and 500 pixels and the webpage will automatically adjust for resolutions at or below this. For example:

      • The title's size reduces based on the screen size and the image sizes scale down to accomodate for the smaller screen.
      • The navigation bar is now at the top of the page and the position is fixed, meaning as you scroll down the nivation bar will stay at the top of your mobile screen, similar to how the 'Back to top' button works.
      • I removed the 'Back to Top' button as users may accidently tap this button which would be rather distruptive.
      • I removed the hyperlinks in the footer to switch between pages as they seemed to not resize properly in the footer which caused some undesirable effects.
      • Additionall, for devices that have a screen size smaller than 500 pixels, the navigation bar will now stack on top of itself. I did this because the words become very squished on a small screen while keeping the font size legible.
